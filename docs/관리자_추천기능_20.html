🔵 A. 매일 여는 화면이 빨라진다

지금 관리자에서 손이 제일 많이 가는 곳들입니다.

1

관리자 홈 (오늘 브리핑)

中

로그인하면 바로: 오늘 몇 팀 · 다음 손님 몇 시 누구 · 미입금 N건 · 환불 대기 N건 — 한 화면.

지금은 로그인하면 7월 달력이 뜹니다. "오늘 뭐 해야 하지?"를 알려면 날짜 누르고 → 테마 탭 4번 누르고 → 입금·환불 탭 또 눌러야 합니다. 하루 5팀만 넘어도 이게 매번 반복됩니다.

참고 Checkfront가 "Daily Manifest"를 별도로 낸 이유가 정확히 이것 · FareHarbor Manifest = "하루 예약의 실시간 스냅샷"

2

날짜별에 "전체" 테마 탭

小

테마 탭 맨 앞에 [전체] 추가 → 그날 4개 테마 손님을 시간순으로 한 번에.

지금 날짜별은 테마를 하나 골라야만 보입니다. 그날 전체를 보려면 탭을 4번 눌러야 하고, 폰에서는 더 답답합니다. 1번(관리자 홈)보다 훨씬 싸고 효과는 절반쯤 됩니다.

이걸 넣으면 예전 "월 전체" 보기가 하던 일이 날짜별 안에서 해결됩니다

3

전화 바로 걸기 / 문자 바로 열기

小

예약 목록의 전화번호를 누르면 폰에서 바로 전화·문자앱이 열림.

지금 전화번호는 그냥 글자입니다. 폰으로 보다가 손님에게 전화하려면 번호를 눈으로 읽고 다시 손으로 찍어야 하고, 오타 나면 엉뚱한 사람에게 전화합니다. 코드 한 줄 수준인데 매일이 바뀝니다.

참고 FareHarbor Manifest는 손님 연락 수단을 현장에서 바로 쓰도록 노출

4

어디서든 검색 (전역 검색바)

小

관리자 맨 위에 검색칸 하나 — 이름·전화 치면 어느 탭에 있든 그 손님 예약이 뜸.

지금 검색은 [예약 › 목록·검색] 안에만 있습니다. 손님 전화를 받는 중에 탭을 옮겨 다녀야 합니다. 전화 응대는 초 단위인데 클릭이 3번입니다.

기존 검색 API 재사용 — 화면만 위로 끌어올리는 일

5

폰으로 새 예약 알림

中

관리자 화면을 안 켜놔도 폰에 새 예약 알림이 뜸.

지금 "🔔 새 예약" 알림은 관리자 화면을 띄워놔야만 옵니다. 사장님은 현장에 계시죠. 그 사이 30분 자동취소가 먼저 돌아 예약이 날아갑니다.

웹 푸시(PWA) — 폰 홈에 관리자 아이콘 추가하면 앱처럼 알림

🟠 B. 지금은 아예 못 하는 일

지금 관리자로는 방법이 없어서 저를 부르거나 손으로 우회해야 하는 것들입니다.

6

예약 옮기기 (날짜·시간 변경)

中

취소·재등록 없이 날짜/시간/인원만 변경. 옮길 칸이 비었는지 자동 검사.

실무에서 제일 흔한 요청이 "한 시간만 미뤄주세요"인데 지금은 취소 후 재등록뿐입니다. 그러면 환불율이 계산되고 환불 큐에 뜨고 입금 상태가 초기화됩니다 — 손님은 그대로 오는데 장부가 "취소 1건 + 신규 1건"으로 더러워집니다.

참고 Resova는 온라인·워크인·단체를 한 화면에서 변경·취소·환불까지

7

테마별 예약금 화면에서 수정

中SQL

지금 "읽기 전용"으로만 보이는 테마별 예약금(3만·2.5만·12만·6.3만)을 직접 고치게.

가격을 바꾸려면 매번 저를 불러야 합니다 — 코드 고치고 배포하고. 설정 화면에 "변경은 개발자에게 요청"이라고 써 있는 게 지금 상태입니다. 가격은 사장님이 정하는 건데 사장님이 못 바꿉니다.

문자 문구에 금액이 들어가므로 같이 바뀌게 묶어야 함(안 그러면 안내 금액이 어긋남)

8

특정 날짜만 미리 열기

中

"이용일 7일 전 저녁 9시" 규칙과 별개로, 특정 날짜·테마만 지금 바로 오픈.

단체·인플루언서·이벤트 문의가 오면 규칙 때문에 온라인으로 못 받고 수동 등록해야 합니다. 지금 설정에는 닫는 도구(휴무·마감)만 있고 여는 도구가 없습니다.

기존 차단 테이블을 반대로 쓰는 방식 — 새 테이블 불필요

9

여러 칸 한 번에 마감 (단체·대관)

小

시간 여러 개를 체크해서 한 번에 마감/해제.

대관·촬영으로 오후를 통째로 비우려면 지금은 칸을 하나씩 마감해야 합니다. 사자의 서는 하루 9칸이라 9번 눌러야 합니다.

휴무일(하루 전체)과 칸 하나 사이가 비어 있음

10

시간표 복사 / 미리보기

小

"금요일 시간표를 토요일로 복사" · 바꾸기 전에 손님 화면에 어떻게 보일지 미리보기.

지금 테마별·요일별 시간표는 칸을 하나씩 손으로 넣습니다. 태초의 신부 주말은 10칸이라 10번입니다. 그리고 저장하면 바로 손님 화면에 나갑니다 — 미리 볼 방법이 없습니다.

시간표는 한 번 잘못 저장하면 그날 예약이 안 들어옴 → 미리보기의 값이 큼

11

문자 실패 재시도 · 발송 내역 검색

小

실패한 문자에 [다시 보내기] 버튼 + 발송 내역을 이름·전화로 검색.

문자가 실패해도 지금은 내역에 빨간 글씨로 보일 뿐 다시 보낼 방법이 없습니다. "저 문자 못 받았어요" 전화가 오면 내역 50건을 눈으로 훑어야 합니다.

발송 로그(sms_log) 테이블은 이미 있음 — 화면만 붙이면 됨

🔴 C. 실수를 막는다

1인 운영이라 실수를 잡아줄 사람이 없습니다. 화면이 대신 잡아야 합니다.

무통장은 통장에 찍힌 이름으로만 대조합니다. 친구·엄마·회사 이름으로 넣는 경우가 흔한데 지금은 메모에 손으로 적는 수밖에 없고, 환불할 때 누구 돈인지 다시 헷갈립니다.

小

예약 변경 이력

中SQL

지금은 접수·확정·취소 시각만 남습니다. "제가 취소한 적 없는데요?" 하면 확인할 방법이 없습니다. 1인이라 "누가"는 필요 없지만 "언제 뭐가"는 필요합니다.

감사(audit)가 아니라 사장님 본인 기억 보조용 — 가볍게

테스트 데이터 정리 도구

🟣 D. 손님을 기억한다

데이터는 이미 쌓이고 있는데 화면에서 안 씁니다. 꺼내 쓰기만 하면 됩니다.

16

손님 카드 (전화번호 기준 이력)

中

예약 상세에 그 번호의 과거 예약·방문·취소 이력 + "3번째 방문" 배지 + "아직 안 한 테마".

전화번호 인덱스는 이미 있는데 화면에서 안 씁니다. 재방문 손님인지 모르고 응대하면 "또 오셨네요" 한마디를 못 합니다. 테마가 4개뿐이라 "이 손님이 아직 안 한 테마"가 보이면 바로 권해드릴 수 있습니다.

참고 캐치테이블은 방문 이력 기반 단골 관리를 앞세움 · Bookeo는 고객 프로필 제공
