# 설정 안내 — 예약·리뷰 작동시키기

> 홈페이지 화면은 바로 보이지만, **예약·리뷰가 실제로 저장되려면 데이터베이스(Supabase) 연결**이 한 번 필요해요.
> 아래는 **사장님(주인님)만 할 수 있는** 계정·가입 작업이에요. 천천히 따라 하시면 됩니다.

---

## 0. 지금 상태
- ✅ 홈·예약·조회/취소·리뷰·비즈니스 **화면 전부 완성** (보기만 하는 건 지금도 됨)
- ⬜ 예약/리뷰를 **저장**하려면 → Supabase 연결 (아래 1번)
- ⬜ 예약금 결제·자동문자 → 그다음 단계 (가입 필요·유료)

---

## 1. Supabase 만들기 (무료, 한 번만)

1. https://supabase.com 가입 → **New project** 생성 (지역: **Northeast Asia (Seoul)** 추천)
2. 프로젝트가 만들어지면 왼쪽 메뉴 **SQL Editor** 열기
3. 이 저장소의 `supabase/schema.sql` 내용을 **복사 → 붙여넣기 → Run** (예약·리뷰 표가 만들어짐)
4. 왼쪽 **Settings → API** 에서 두 값 복사:
   - **Project URL** (예: `https://xxxx.supabase.co`)
   - **service_role** 키 (⚠️ 비밀키 — 외부 공개 금지)

## 2. 내 컴퓨터에서 연결 (로컬 테스트)
1. 저장소의 `.env.local.example` 를 복사해 **`.env.local`** 파일로 만들기
2. 위에서 복사한 값 채우기:
   ```
   SUPABASE_URL=https://xxxx.supabase.co
   SUPABASE_SERVICE_ROLE_KEY=eyJhbGci...
   ```
3. `npm run dev` 다시 실행 → 예약·리뷰가 진짜로 저장됨

## 3. 인터넷에 공개 (Vercel 배포, 무료)
1. https://vercel.com 에 GitHub 계정으로 로그인 → 이 저장소 **Import**
2. **Environment Variables** 에 위 두 값(`SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`) 등록
3. Deploy → 인터넷 주소가 생깁니다
