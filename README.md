## 2026-07-16 — 🚀 테마 포스터 2,014KB → 94KB (95% 감소)

> 홈 화면이 내려받는 이미지 2,065KB 중 **2,014KB(97%)가 포스터 4장**이었음.

- **원인**: 포스터를 **CSS 배경**(`style={{backgroundImage}}`)으로 넣어서 **next/image 최적화를 통째로 건너뜀**. 브라우저가 원본을 그대로 받음.
  · 같은 페이지의 **매장 지도는 `<Image>` 라 1.97MB → 28KB WebP**(70배 차이)로 잘 내려가고 있었음 — 증거가 같은 화면 안에 있었던 셈.
  · 포스터 원본 **726×1024** 인데 화면엔 **266×399** 로 표시 → 필요한 픽셀의 약 7배. 게다가 락다운시티는 사진인데 **PNG**(사진에 최악)라 1MB.
- **수정**: `<Image fill sizes="(max-width:520px) 82vw, (max-width:980px) 45vw, 280px">` + `.thumb-img{object-fit:cover}`. 파일은 그대로 두고 **넣는 방식만** 바꿈.
- **결과(실측)**: `poster-ldc.png` 1,057KB → **25KB** · `poster-duat.png` 711KB → **18KB** · `poster-bride.jpg` 164KB → **36KB** · `poster-time.jpg` 82KB → **15KB**. **합계 2,014KB → 94KB.** 전부 WebP·384w 로 자동 변환됨(고해상도 화면은 srcset 으로 더 큰 걸 받음).
- **왜 예전엔 배경이었나**: `<img>` + 호버 확대가 카드 밖으로 삐져나와서 배경으로 바꿨던 것. **그 버그는 `.tcard{overflow:clip}` 으로 이미 막혀 있어** 되돌려도 안전 — 호버 확대 정상 동작 확인.
- 체감은 **모바일에서 큼**: LTE 손님이 테마 목록 보려고 2MB 받던 것이 94KB.
