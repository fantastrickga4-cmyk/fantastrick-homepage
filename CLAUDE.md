# CLAUDE.md — 이 프로젝트에서 Claude가 지켜야 할 약속

> 이 파일은 **Claude(AI 도우미)** 가 매번 읽는 사용 설명서예요.
> 사람이 봐도 되지만, 주로 AI가 "이 프로젝트는 이렇게 일해야지" 하고 기준으로 삼는 문서입니다.

---

## 1. 이 프로젝트가 뭐예요? (한 줄 요약)

**판타스트릭(Fantastrick) 홈페이지**를 만드는 프로젝트예요.
회사/브랜드를 소개하고, 방문자가 보기 좋고 쓰기 편한 **웹사이트**를 직접 만드는 거예요.

목표 기능(시작점 — 진행하며 구체화):
- 🏠 브랜드/회사 소개 (메인 페이지, 소개, 서비스/제품 안내)
- 📱 PC·태블릿·스마트폰 어디서 봐도 깨지지 않는 **반응형** 디자인
- ✉️ 방문자가 연락할 수 있는 문의/연락 수단
- ⚡ 빠르고 가볍게 — 보기 좋은 디자인 + 빠른 로딩

> 📌 구체적인 페이지 구성·디자인 방향·기술 스택은 사용자와 정해가며 이 문서에 기록한다.

---

## 2. 사용자(주인님)에 대해 — ⭐ 매우 중요

- 사용자는 **개발 입문자("초등학생 개발자")** 예요.
- **어려운 전문 용어를 쓰면 못 알아들어요.** 항상 **쉬운 한국어**로 설명!
- 전문 용어가 꼭 필요하면 → 괄호로 풀이를 붙여요. 예: `커밋(저장)`, `푸시(인터넷에 올리기)`, `반응형(화면 크기에 맞춰 모양이 바뀌는 것)`, `배포(인터넷에 홈페이지를 올려 누구나 보게 하는 것)`.
- 시간이 거의 없어요 → **"이거 할까요 저거 할까요" 묻지 말고**, 제일 좋은 방법을 **알아서 골라서 바로 진행**.
- 단, **돈이 드는(유료) 서비스**가 필요할 때 **그때만 미리 알려주기**.

---

## 3. 일하는 방식 (Claude 행동 규칙)

0. **콜드 스타트(시작할 때마다)**: 새 대화를 켜면 **제일 먼저** 이 프로젝트 폴더의 `CLAUDE.md`(이 약속) + `README.md`(변경 기록) + `DIRECTORY_MAP.md`(폴더 지도)를 읽고, "지금 어디까지 했는지 / 다음 할 일이 뭔지"를 파악한 뒤 이어서 일한다. (시키지 않아도 자동으로)
1. **묻지 않고 진행**: 합리적인 기본값으로 알아서 결정. 정말 사용자 본인만 할 수 있는 일(예: 본인 계정 로그인, 도메인 구매)만 부탁.
2. **유료만 알림**: 무료로 되는 건 그냥 진행. 결제가 필요하면(도메인, 유료 호스팅, 유료 폰트/이미지 등) 먼저 알려주기.
3. **꼼꼼히 기록**: 무엇을 바꿨는지 항상 `README.md`(변경 기록)에 적기.
4. **폴더 지도 관리**: 폴더/파일이 바뀌면 `DIRECTORY_MAP.md`(폴더 지도) 업데이트.
5. **쉬운 보고**: 작업이 끝나면 "무엇을 / 왜 했는지"를 짧고 쉽게 알려주기.
6. **메이저 작업 완료 시 자동 확인(미리보기)**: 큰 작업(페이지 완성, 디자인 큰 변경 등)이 끝나면 시키지 않아도 **실제 브라우저(크롬)로 직접 열어 화면을 확인**하고, 깨진 곳·이상한 곳이 보이면 **알아서 고친다.** 필요하면 화면 캡처를 보고에 첨부한다.
   - Chrome 위치: `C:\Program Files\Google\Chrome\Application\chrome.exe`
   - 화면 확인이 필요하면 Playwright 같은 도구를 `channel:'chrome'`로 설치된 크롬을 쓰도록 설정해 사용.
7. **⚠️ 중요 파일·대량 삭제 시 먼저 알림 (꼭!)**: 아래 경우엔 **그냥 지우지 말고**, "무엇을 / 왜 / 몇 개" 지우는지 **먼저 사용자에게 알리고 확인**받은 뒤 진행한다. (실수로 자료를 잃지 않게)
   - **중요 파일**을 삭제하거나 덮어쓸 때: 소스 본체(HTML·CSS·JS 페이지 파일), 사용자가 준 **원본 자료**(로고·이미지·문구·브랜드 자산), 문서(`README.md`, `CLAUDE.md`, `DIRECTORY_MAP.md`), 설정 파일(`package.json` 등)
   - **파일 대량 삭제**: 한 번에 여러 개(대략 3개 이상)나 폴더 통째로 지울 때
   - 단, **자기가 만든 임시/빌드 결과물**(예: `node_modules`, `dist`, 캐시, 테스트용 더미)을 스스로 정리하는 것은 알림 없이 진행해도 됨. 단 "임시파일 N개 정리함"처럼 **결과는 보고**한다.
8. **디자인 일관성**: 한 번 정한 색·폰트·간격·버튼 모양은 **전체 페이지에서 통일**되게 유지한다(가능하면 CSS 변수로 한 곳에서 관리). 새 페이지를 만들 때 기존 스타일을 먼저 본다.

## 4. 저장(커밋) & 올리기(푸시) 규칙 — 자동

사용자가 **자동 저장+올리기를 미리 허락**했어요. 매번 다시 묻지 않아요.

| 상황 | 행동 |
|------|------|
| **중요한 변경** (페이지 추가, 큰 디자인/구조 변경 등) | 🔴 **바로** 커밋(저장) + 푸시(올리기) |
| **자잘한 변경** (오타, 색 미세조정, 작은 문구 수정 등) | 🟡 **10번쯤 모이면** 한 번에 커밋 + 푸시 |

- 커밋 메시지는 **한국어로 간단하게**.
- 아직 인터넷 저장소(GitHub)가 연결 안 됐으면 → 일단 **로컬(내 컴퓨터)에만 저장**하고, 연결은 사용자에게 1회만 부탁.

> 📌 자잘한 변경 누적 횟수는 아래 "작업 카운터"에서 셉니다.

### 작업 카운터 (자잘한 변경 모음)
- 마지막 푸시 이후 자잘한 변경 횟수: **0 / 10**
- (10이 되면 커밋+푸시하고 다시 0으로)

## 7. 기술 메모 (Claude가 이 환경에서 겪은 함정 — 꼭 기억!)

1. **PowerShell 스크립트(.ps1)·배치(.bat)는 한글이 들어가면 인코딩 주의**. 한글이 깨지면 "UTF-8 + BOM"으로 다시 저장:
   ```powershell
   $f='경로'; $b=[IO.File]::ReadAllBytes($f)
   if($b.Length-ge3-and$b[0]-eq0xEF){$b=$b[3..($b.Length-1)]}
   [IO.File]::WriteAllText($f,[Text.Encoding]::UTF8.GetString($b),(New-Object Text.UTF8Encoding($true)))
   ```
2. **Write/Edit 도구는 BOM 없이 저장**하므로, 한글 .ps1/.bat을 만든 뒤엔 위 방법으로 BOM을 다시 입힐 것.
3. **이 Claude 도구 셸에서는 UAC(관리자 권한) 창이 안 뜨고 자동 거부됨.** 관리자 권한이 필요한 작업은 **사용자에게 부탁**한다("윈도우키 → powershell → 우클릭 → 관리자 권한으로 실행 → 한 줄 붙여넣기"). 홈페이지 작업은 보통 관리자 권한이 필요 없음.
4. **웹 페이지 한글 깨짐 방지**: HTML 파일 맨 위 `<head>`에 항상 `<meta charset="utf-8">` 넣기.
